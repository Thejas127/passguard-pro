// password guide

Build a Strong
Secure Password

📏
Use 12–20+ characters
Length is the single biggest strength factor. Each extra character multiplies difficulty exponentially — aim for 16 or more.
🎲
Mix all 4 character types
Combine uppercase, lowercase, numbers, and symbols. Each type expands the possible character pool from 26 to 94 — a massive jump.
🔀
Use a passphrase
4–5 unrelated random words like Cloud$Mango#Brick7 are memorable and have extremely high entropy.
🔑
Use a password manager
Bitwarden, 1Password, or KeePass generate and store unique, random passwords for every site — you only remember one master password.
📱
Enable 2FA everywhere
Two-factor authentication adds a second layer. Even if your password leaks, attackers can't get in without your physical device.
🔄
Change after any breach
Monitor haveibeenpwned.com. If a site you use is compromised, change that specific password immediately — not all your passwords.
📖
No dictionary words
Words like "sunshine", "dragon", or "monkey" top every wordlist. Cracking tools test millions of words before brute force — found in seconds.
👤
No personal information
Avoid names, birthdays, anniversaries, pet names, phone numbers, or favorite teams — attackers research targets before trying passwords.
🔁
Never reuse passwords
Credential stuffing attacks take leaked passwords and try them on every major site. One breach exposes all your accounts if you reuse.
🔢
Avoid keyboard patterns
Sequences like qwerty, 123456, or Password123! are the first things any cracking tool tries — regardless of length.
📋
Don't store in plain text
Sticky notes, the Notes app, or an unencrypted spreadsheet are major vulnerabilities. Use an encrypted password manager instead.
🔗
Leet speak alone isn't enough
Replacing letters like p@$$w0rd is in every cracking ruleset. It adds minimal real entropy — pair it with length and randomness.

Step 01
Define your character pool
The pool is how many unique characters are possible. Each type you add expands it: Lowercase only: 26 chars + Uppercase: 52 chars + Numbers: 62 chars + Symbols (~32): 94 chars
Step 02
Calculate entropy (bits)
Entropy measures true randomness. More bits = exponentially harder to crack. Entropy = Length × log₂(Pool) Example: 16 chars, all types = 16 × log₂(94) ≈ 105 bits
Step 03
Estimate crack time
Assuming a GPU cluster at 10 billion guesses/sec: Combinations = 2^entropy Time = Combinations ÷ 10¹⁰ 60 bits → ~37 years 80 bits → ~38 million years 100 bits → practically impossible
Step 04
The golden targets
Aim for at least 80 bits for everyday accounts. Use 100+ bits for banking, email, and password managers. Length always contributes more than complexity alone.
